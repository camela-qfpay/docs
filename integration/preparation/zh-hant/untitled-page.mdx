# API 環境介紹

QFPay API 提供三個主要環境：**正式環境（Production）**、**線上測試環境（Live Testing）** 與 **沙盒環境（Sandbox）**。

每個環境用途不同，且對應不同的 base URL。請根據實際需求選擇合適的環境進行測試或部署。

## 環境說明

| 環境類型       | Base URL                             | 說明                                                                 |
| ---------------- | -------------------------------------- | ---------------------------------------------------------------------- |
| 正式環境       | `https://openapi-hk.qfapi.com`       | 實際交易，會結算並記入對帳單                                       |
| 線上測試環境   | `https://test-openapi-hk.qfapi.com`  | 真實交易流程，但若綁定測試帳戶則不會結算                           |
| 沙盒環境       | `https://openapi-int.qfapi.com`      | 僅供模擬信用卡交易，不會扣除任何實際金額                            |

<Warning>
若在 **線上測試環境** 中使用測試帳戶進行交易，該筆交易將 **不會被結算**。請務必於測試完成後立即執行退款，以免造成混淆。
</Warning>

<Note>
切換環境時，請確認使用正確的網域（Base URL）。不同環境的憑證與端點不可混用，否則將導致簽名驗證或授權錯誤。
</Note>
