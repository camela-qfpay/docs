為了確保平台的公平使用與最佳效能，我們實施了 API 請求頻率限制政策。本頁說明請求限制細節及相關處理方式。

## 請求頻率限制

- **限制值**：每個商戶每秒最多可發送 **100 筆請求（RPS）**，每分鐘最多 **400 筆請求**。
- **超出限制的回應**：若超過上述限制，API 將回應 **HTTP 429 Too Many Requests** 狀態碼。

## 使用建議

為避免觸發限制並有效使用 API，建議遵循以下最佳實踐：

1. **批次處理請求**：在可能情況下使用批次處理，減少個別請求數量。
2. **優化資料查詢**：合理設計查詢條件，避免過度頻繁請求；可使用篩選與分頁功能。
3. **實作快取**：對重複性高的查詢實施快取策略，減少重複請求。
4. **監控使用量**：追蹤請求頻率，並記錄日誌以便後續分析與調整。

## 錯誤處理建議

若觸發頻率限制，請參考以下建議進行應對：

1. **處理 HTTP 429 錯誤**：
   - 在應用程式中設計重試邏輯，例如：
     - 停止請求一段時間（如延遲幾秒再重試）
     - 記錄錯誤發生時間與頻率，以利後續監控
2. **實作指數退避策略（Exponential Backoff）**：
   - 每次重試時逐步延長等待時間，避免立即再次造成阻塞。
